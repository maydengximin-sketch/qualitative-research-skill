# 07 · 报告写作：COREQ、引语呈现与章节组织

> 投稿/送审前先按报告规范自检。质性研究投稿被拒最常见原因之一是**方法透明度不足**——规范清单就是用来堵这个漏洞的。

## 一、报告规范怎么选

| 规范 | 适用 | 项数 |
| --- | --- | --- |
| **COREQ** | 访谈与焦点小组研究 | 32 项，3 大块 |
| **SRQR** | 各类质性研究通用 | 21 项 |
| ENTREQ | 质性证据综合（meta-synthesis） | 21 项 |

→ 访谈/焦点小组优先用 COREQ；用 `templates/coreq-checklist.md` 逐项核。

### COREQ 三大块（32 项）
1. **研究团队与反身性**：访谈者身份、资质、与受访者关系、研究者预设。
2. **研究设计**：方法学取向、抽样、样本、场景、数据收集（提纲、录音、时长、是否饱和、是否返还转录稿）。
3. **分析与发现**：编码者数量、编码树来源、软件、是否成员校验、引语标注、数据与发现一致性。

## 二、引语（Quotations）呈现规范

引语是质性研究的**证据**，呈现要讲究：
- **每个主题配 1–3 条**代表性引语，太多显得堆砌，太少缺支撑。
- **标注来源**：受访者代号（P07）、必要的情境（如年龄/角色），确保不可反推身份。
- **区分排版**：长引语独立成段缩进，短引语嵌入正文加引号。
- **保真但可清理**：可删"嗯/那个"等语气词（注明做了清洁），用 […] 标省略，用 [ ] 补必要语境。
- **避免 cherry-picking**：所选引语要能代表该主题在数据中的分布，必要时报该主题被多少受访者提及。
- **呈现反例（negative/deviant case）**：主动展示不符合主流模式的声音，增强可信性。
- **分析 ≠ 复述引语**：引语后必须有研究者的解释/论证，不能只贴引语让其"自己说话"。

### 引语呈现模板
> **主题 1：在不确定中"先撑住再说"**
> 多数受访者（9/12）描述了一种延迟判断、优先维持现状的应对方式。如 P03 所说：
>> "我也不知道对不对，反正……先撑住再说吧，走一步看一步。"（P03，34 岁，个体经营者）
>
> 这种"撑住"并非被动，而是一种主动的风险管理策略——它把……（研究者分析）

## 三、发现/讨论章节组织

### 发现（Findings/Results）
- 按**主题/范畴**组织，而非按访谈对象逐个复述。
- 每个主题：主题陈述句 → 展开（含子主题）→ 引语证据 → 研究者分析。
- 扎根理论：呈现核心范畴 + 概念模型图 + 范畴间关系命题。
- 用主题图/模型图可视化结构（可用 draw.io、NVivo concept map、Graphviz）。

### 讨论（Discussion）
- 把发现**对话**到已有文献：印证了什么、挑战了什么、补充了什么。
- 提炼理论贡献与实践启示。
- 诚实交代局限（样本、情境、研究者立场）与未来方向。

## 四、方法章节写作骨架（可套用）
1. 研究设计与范式（呼应 `01`）
2. 抽样与参与者（呼应 `02`，附匿名化样本表）
3. 数据收集（提纲设计、过程、转录、伦理；呼应 `03`）
4. 数据分析（方法学、编码流程、软件、谁编码、如何达成一致；呼应 `04`/`05`）
5. 信实度/信度（ICR 或四标准、饱和、反身性；呼应 `06`）

## 五、投稿前终检清单
- [ ] 填完 COREQ/SRQR 并作为附件
- [ ] 方法学名称与编码语言全文一致（无流派混用）
- [ ] 范式 ↔ 信度策略 ↔ 是否报 Kappa 三者自洽
- [ ] 每个主题有引语证据且有分析，含反例
- [ ] 饱和有证据、抽样有论证
- [ ] 有反身性/立场声明与伦理声明
- [ ] 概念模型图清晰、与正文一致
