# 09 · 网络文本与多模态数据

> 当数据来自**社交媒体**（小红书/微博/大众点评/B站）或为**音视频、图片**时，编码方法仍是扎根/主题/内容分析，但**采集、清洗、伦理、多模态编码**有专门要求。这类数据近年极热门，适合形象建构、消费行为、用户体验、话语分析等选题。

## 一、网络文本数据（社交媒体）

### 采集流程（以小红书为例）
1. **确定检索词与范围**：围绕主题定关键词，明确平台、时间范围、内容类型（笔记正文/评论）。
2. **纳入排除标准**：如"正文≥X 字、与主题相关、非广告硬推"，并记录筛选规则。
3. **采集字段**：正文、发布时间、点赞/收藏/评论数、（必要时）配图、热门评论。
4. **清洗**：去 emoji 乱码、广告、重复；话题标签（#xxx）可单独提取作"自带标签"分析；@提及清理。
5. **匿名化**：用户名替换为编号（U001…），去除可识别信息。
6. 整理为规范文件（每条一段，或 Excel 一行一条）导入 NVivo/MAXQDA。

### 采集字段与用途
| 字段 | 用途 |
| --- | --- |
| 正文 | 编码与主题分析主体 |
| 发布时间 | 时间趋势/阶段对比 |
| 点赞/收藏/评论数 | 衡量热度、筛典型样本 |
| 配图/视频 | 多模态分析（视觉形象） |
| 热门评论 | 受众反应、互动话语 |

### 伦理与合规（务必写进方法/局限）
- **只采集公开可见信息**，不抓取私密/需登录授权内容，不商用。
- 匿名化处理，必要时对引用的原帖做改写以防反向检索定位个人。
- 论文表述："数据来自平台公开发布内容，已做匿名化处理。"
- 注意平台用户协议与当地数据法规；机构有 IRB 的先过审。

## 二、多模态数据（音频/视频/图片）

- **音频/视频**：可在时间轴上直接对片段编码，也可导入转录稿与媒体关联，边听边编；适合访谈录音、课堂实录、短视频研究。报告时区分"基于转录的编码"与"基于视听的编码"。
- **图片/视觉**：在图片上框选区域编码并加文字描述；适合视觉形象、海报、版面、配图分析。可借鉴视觉社会符号学（Kress & van Leeuwen）的再现/互动/构图三层。
- **多模态整合**：文本主题 + 视觉意象 + 互动数据三角印证（如小红书"文案建构 + 配图视觉 + 评论反应"）。

### 实操注意
- 媒体文件大，导入用"**仅链接不嵌入**"，避免 NVivo 项目膨胀到数 GB。
- 视频编码工作量大，建议先转录再精编，纯视听编码仅用于语言之外的信息（表情、动作、画面）。

## 三、宏观—微观结合（大数据量时）
文本量上千条时，先用 **ROST CM6 / Python(jieba)** 做分词、词频、情感、语义网络的"宏观扫描"，定位重点，再抽典型样本导入 NVivo 做"微观深描"。宏观（频率分布）+ 微观（意义解读）结合，论文层次更立体（详见 `10` 混合方法与 `11` 软件实操）。
