# 03 · 数据收集：访谈、焦点小组、转录与伦理

> 生成访谈提纲前，**先问清楚**：研究问题、方法学、受访对象、访谈时长预期。提纲必须服务于 RQ，且与方法学一致（现象学问体验、扎根理论问过程）。

## 一、半结构访谈提纲设计

### 提纲分层结构（推荐）
1. **暖场/背景问题**：建立信任、收集情境信息（"先聊聊你平时……"）。
2. **核心问题**：每个对应一个子 RQ，开放、不诱导。
3. **探询（probes）**：在每个核心问题下预设 2–3 个跟进点（"能举个例子吗？""当时你的感受是？""后来呢？"）。
4. **收尾问题**："还有什么我没问到但你觉得重要的？"

### 提问质量原则
- 开放优先：用"如何/怎样/能不能讲讲"，避免是非题与诱导。
- 一次一问，避免双重问题。
- 用受访者的语言，避免学术黑话。
- 多用具体经历（"上一次……是怎样的"）而非抽象观点（"你觉得 X 重要吗"）。
- 中立追问，不评判、不补全受访者的话。

### 好问题 vs 差问题
| 差 | 好 |
| --- | --- |
| "你是不是因为方便才用它？"（诱导+是非） | "你一般在什么情况下会用它？" |
| "你觉得它有哪些优缺点？"（抽象、双重） | "能讲讲最近一次让你印象深刻的使用经历吗？" |
| "你满意吗？" | "如果让你向朋友形容这段体验，你会怎么说？" |

→ 可直接套用 `templates/interview-guide-template.md`。

## 二、焦点小组（Focus Group）

- **何时用**：研究的是群体互动、规范、共识/分歧的协商，而非个人深层隐私经验。
- **规模**：每组 6–10 人（敏感话题 4–6 人），3–6 组常可达饱和。
- **同质 vs 异质**：组内通常同质（便于开口），组间求差异。
- **主持要点**：主持人引导而非主导；设计开场→过渡→核心→结束的问题流；用"还有不同看法吗？"激发互动；管理强势发言者。
- **记录**：需录音 + 助理记录座位图与非语言/谁说了什么（多人难辨声）。
- **分析单位**：注意区分"个体观点"与"小组层面的互动/共识"，二者都要分析。

## 三、转录（Transcription）

| 规范 | 适用 | 要点 |
| --- | --- | --- |
| 逐字转录 Verbatim | 多数主题/扎根分析 | 含"嗯""停顿"，保留口语 |
| 清洁逐字 Clean verbatim | 政策/内容分析 | 去除无意义语气词 |
| Jefferson 系统 | 会话分析/话语分析 | 标注重叠、停顿时长、语调、重音 |

实操建议：
- 统一编号规则（如 P01、FG2-P3）和时间戳。
- 标注说话人、停顿、笑声、[听不清]。
- 匿名化：转录阶段即替换真实姓名/单位/地名为代号，单独保存对照表。
- AI 自动转录（讯飞、通义听悟等）可加速，但**必须人工逐句校对**，尤其方言、专有名词。

## 四、伦理与知情同意（必做）

- 知情同意：研究目的、自愿、可随时退出、保密、数据用途与保存、风险。→ 见 `templates/consent-form-template.md`
- 匿名化与数据安全：去标识、加密存储、限定保存期限。
- 敏感人群（未成年人、患者、弱势群体）需监护人同意/伦理审批（IRB/伦理委员会）。
- 涉及盗版/灰色/违法情境的数据（如非法平台用户评论）需特别的伦理声明：只用公开数据、不诱导违法、不可识别个人。
- 报告中引语要确保无法反推身份。

## 五、数据收集质量自检
- [ ] 提纲每个问题都能溯源到某个 RQ
- [ ] 做过 1–2 次预访谈（pilot）并据此修订提纲
- [ ] 录音设备双备份
- [ ] 知情同意已签署/记录
- [ ] 访谈后即写田野备忘录（情境、印象、初步想法）
- [ ] 匿名对照表与数据分开加密保存
