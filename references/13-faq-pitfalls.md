# 13 · 避坑指南与答辩高频问题

> 给用户做"临门一脚"的支持：写作/答辩前的自查与应答。引用具体标准时回到对应 reference。

## 一、新手十大常见错误
1. 没想清研究问题就猛编码，建出几百个混乱节点。
2. 开放编码用大词、抽象词，脱离原话（应贴近原话，见 `04`）。
3. 只编码不写备忘录，思考过程无审计追踪。
4. 节点结构混乱，主轴与开放混在一起。
5. 完全依赖软件自动/AI 编码，被质疑信度（见 `11`）。
6. 不分范式无差别要求 Kappa，或建构主义研究硬报 Kappa（见 `06`）。
7. 把节点清单当结果直接搬进论文，缺少解释与引语。
8. 引语不标来源、不匿名、只挑支持性证据（cherry-picking，应呈现反例）。
9. 只说"访谈 N 人达到饱和"而无追踪证据（见 `06`）。
10. 数据不备份、过程无记录，无法复现。

## 二、答辩/评审高频问题应答
| 可能被问 | 应答要点 |
| --- | --- |
| 为什么用质性方法？ | RQ 是"怎么样/为什么"，需理解机制而非测量频率 |
| 样本这么小有代表性吗？ | 质性追求信息饱和而非统计代表性，已给饱和证据 |
| 编码是你主观的吗？ | 有编码本 + 反身性；视范式补双人编码/Kappa 或信实度策略 |
| 软件是不是自动出结论？ | 软件仅辅助管理/统计，编码与解释由研究者完成 |
| 怎么保证可靠性？ | 按范式：信实度四标准 / 三角验证 / 审计追踪 / Kappa |
| 怎么从编码到理论？ | 三级编码层层抽象，核心范畴串故事线成模型（见 `04`） |
| 为什么不报 Kappa？（建构主义） | ICR 与建构主义认识论有张力，改用信实度论证（见 `06`） |

## 三、投稿前对照报告规范
- 访谈/焦点小组研究 → COREQ 32 项（见 `07` 与 `templates/coreq-checklist.md`）。
- 一般质性研究 → SRQR 21 项。
- 混合方法 → 额外画设计流程图、做联合展示表（见 `10`）。

## 四、范式—方法—质量标准 自洽性自检
> 三者必须一致，这是评审看穿"假质性"的关键。
- [ ] 范式（建构/后实证/批判/实用）已明确
- [ ] 方法学与范式匹配（如建构主义扎根 → Charmaz）
- [ ] 质量标准与范式匹配（建构主义→信实度；后实证/内容分析→ICR）
- [ ] 分析深度与目标匹配（描述→主题/内容；建理论→完整扎根）
- [ ] 引语—主题—结论链条可追溯
