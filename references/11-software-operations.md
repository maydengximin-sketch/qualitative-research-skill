# 11 · NVivo / MAXQDA 实操详解

> 软件是"管理思考的工具"，不是"代替思考的工具"。编码、归类、解释由研究者完成，软件负责记录、组织、检索、可视化、量化。本篇给 NVivo 为主、MAXQDA 对照的关键操作。Kappa/编码比较见 `06`。

## 一、NVivo 核心概念对照
| 中文 | English | 含义 |
| --- | --- | --- |
| 节点/编码 | Node / Code | 主题/概念容器，编码的归集点 |
| 案例 | Case | 分析单位（受访者/政策/笔记） |
| 分类 | Classification | 对案例或节点的分组 |
| 属性 | Attribute | 案例的特征变量（性别、年份等） |
| 参考点 | Reference | 某节点被编码的次数/片段数 |
| 覆盖率 | Coverage | 编码片段占整篇的百分比 |
| 备忘录 | Memo | 研究者的分析笔记 |
| 矩阵编码查询 | Matrix Coding Query | 生成行列交叉频次表 |

## 二、标准操作流程
1. **建项目**：纯英文路径，开自动保存；定期"另存为"做带日期的版本备份（NVivo 项目损坏几乎无法修复）。
2. **导入资料**：文档/PDF/媒体/图片/电子表格；音视频用"仅链接不嵌入"控制体积；扫描件 PDF 需先 OCR。
3. **设案例与属性**：把每个受访者/政策/笔记设为案例，用 Excel"分类表"批量赋属性（性别、年龄、机关、年份…）——这是后续交叉分析的前提，**别等编完码才发现没设属性**。
4. **建节点 + 三级编码**：开放（逐句贴近原话）→ 主轴（拖拽归并为父节点）→ 选择（核心范畴 + 模型），见 `04`。
5. **写备忘录/注释**：保留审计追踪（audit trail）。

## 三、查询与可视化（出论文图表）
| 功能 | 用途 | 论文用途 |
| --- | --- | --- |
| 词频查询 + 词云 | 高频词 | 现象描述/引入；中文需设停用词、最小词长 |
| 文本搜索 + 词树 | 看某词的上下文语境 | 话语分析 |
| 矩阵编码查询 | 属性×节点 / 工具×主题 交叉频次 | 政策分析、群体差异对比 |
| 编码比较查询 | 编码者间一致性(Kappa) | 信度（见 `06`） |
| 层次图/旭日图 | 各节点编码量分布 | 结果概览 |
| 概念图/思维导图 | 手动画理论模型 | 结论模型图 |

> NVivo 自带图美观度有限，建议导出数据/图片后用 PPT、Excel、Visio 重绘精修。

## 四、自动编码与 AI 辅助（慎用、必复核）
- 按结构自动编码（按说话人/标题）：适合规范格式的访谈稿。
- NVivo 15 的 AI 自动总结/建议主题：只能当"草稿"，所有结果人工逐条复核。完全依赖自动编码会被质疑信度。
- 答辩口径：核心编码由研究者人工完成，软件仅辅助管理与统计。

## 五、MAXQDA / ATLAS.ti 对照
- MAXQDA：`分析 → 编码者间一致性`（可按段落/句子设容差）；可视化工具丰富（代码矩阵浏览器、文档画像）。
- ATLAS.ti：网络视图（network）强，适合关系建模。
- 三者编码逻辑一致，本 skill 的方法全部通用；纯手工（Word 高亮 + Excel 码表）也能完成，只是检索与计量更费力。
